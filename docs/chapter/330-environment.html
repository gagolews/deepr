In the first part of our book, we have discussed quite a few basic types of objects: numeric, logical, and character vectors, lists (generic vectors), and functions.

Environments, which we introduce in this chapter, are instances of recursive types (compare Figure¬†1). We rarely interact with them directly (unless we need a hash table-like data structure with quick element look up by name).

Still, they are utterly important as they form the basis of the environment model of evaluation which governs how R expressions are computed; see Section¬†17.1.

Important

Each object of type environment consists of:

  • a frame1 ‚Äì stores a set of bindings, which associate variable names with their corresponding values; it can be thought of as a container of named R objects of any type;

  • a pointer to an enclosing environment2.

16.1. üöß Frames: Environments as Object ContainersÔÉÅ

To create a new, empty environment, we call the new.env function:

(e1 <- new.env())
## <environment: 0x55d287966f08>
typeof(e1)
## [1] "environment"

In this section, we treat environments merely as containers for named objects of any kind, i.e., deal with the frame part thereof.

e1[["x"]] <- "x in e1"
e1[["y"]] <- 1:3
e1[["x"]]
## [1] "x in e1"
e1[["spam"]]
## NULL
e1[["y"]] <- e1[["y"]]*10
e1[["z"]] <- NULL  # unlike in the case of lists, creates a new element
names(e1)  # but is not equipped with the `names` attribute
## [1] "x" "y" "z"
length(e1)
## [1] 3
print(e1)  # same with str(e1)
## <environment: 0x55d287966f08>
for (key in names(e1))  { # for (elem in e1) does not work...
    cat(sprintf("%s: ", key))
    str(e1[[key]])
}
## x:  chr "x in e1"
## y:  num [1:3] 10 20 30
## z:  NULL
e1[c("x", "y")]  # but see the mget function
## Error in e1[c("x", "y")]: object of type 'environment' is not subsettable
e1[[1]] <- "bad key"
## Error in e1[[1]] <- "bad key": wrong args for environment subassignment

16.1.2. üöß Pass-by-value, Copy on Demand ‚Äì Not for EnvironmentsÔÉÅ

Environments are the only R objects that are not passed-by-value!

mod <- function(x, key)
    (x[[key]] <- x[[key]]+1)
x <- list(a=1)
mod(x, "a")  # pass a "copy" of x
## [1] 2
x[["a"]]  # state of x did not change
## [1] 1
x <- as.environment(list(a=1))
mod(x, "a")  # pass by "reference"
## [1] 2
x[["a"]]   # x has changed
## [1] 2
y <- x
mod(y, "a")  # pass by "reference"
## [1] 3
x[["a"]]   # x has changed (names `y` and `x` are bound to the same object)
## [1] 3
print(x)
## <environment: 0x55d286acac10>
print(y)
## <environment: 0x55d286acac10>

copy on demand:

n <- 100000000  # like, a lot
t0 <- proc.time()
x <- numeric(n)
proc.time() - t0
##    user  system elapsed 
##   0.108   0.216   0.324
t0 <- proc.time()
y <- x
proc.time() - t0
##    user  system elapsed 
##   0.001   0.000   0.001
t0 <- proc.time()
y[1] <- 1
proc.time() - t0
##    user  system elapsed 
##   0.160   0.200   0.359
t0 <- proc.time()
y[2] <- 2
proc.time() - t0
##    user  system elapsed 
##   0.001   0.000   0.001
